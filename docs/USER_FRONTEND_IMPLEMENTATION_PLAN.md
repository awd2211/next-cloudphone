## 🗓️ 分阶段实施计划

### 方案 A: 单人开发 (推荐度: ⭐⭐)

**适用场景:** 资源有限，只有 1 名前端开发

```
总时长: 约 7-8 周 (包含测试和调整)

Week 1-2: P0 核心功能 (9 天 + 调试)
Week 3-4: P1 官网营销 (10 天 + 调试)
Week 5-6: P1 用户体验 (10 天 + 调试)
Week 7-8: 集成测试、优化、部署
```

**优点:**
- ✅ 代码风格统一
- ✅ 沟通成本低

**缺点:**
- ❌ 时间线太长 (2个月)
- ❌ 无法并行开发
- ❌ 风险集中（单点故障）

---

## ⚠️ 风险与应对

## 📚 附录

### A. API 需求清单

**后端需要提供的新 API:**

1. **忘记密码:**
   - `POST /auth/forgot-password` - 发送重置链接
   - `POST /auth/reset-password` - 重置密码
   - `GET /auth/verify-reset-token/:token` - 验证 token

2. **安全中心:**
   - `GET /security/login-history` - 登录历史
   - `GET /security/sessions` - 活跃会话列表
   - `DELETE /security/sessions/:id` - 强制下线会话
   - `POST /security/2fa/enable` - 启用 2FA
   - `POST /security/2fa/disable` - 禁用 2FA

3. **设备批量操作:**
   - `POST /devices/batch/start` - 批量启动
   - `POST /devices/batch/stop` - 批量停止
   - `POST /devices/batch/delete` - 批量删除
   - `POST /devices/batch/install-app` - 批量安装应用

4. **已安装应用:**
   - `GET /devices/:id/installed-apps` - 已安装应用列表
   - `DELETE /devices/:id/apps/:appId` - 卸载应用
   - `PUT /devices/:id/apps/:appId/update` - 更新应用

5. **设备模板:**
   - `GET /device-templates` - 模板列表
   - `POST /device-templates` - 创建模板
   - `POST /device-templates/:id/create-device` - 使用模板创建设备
   - `DELETE /device-templates/:id` - 删除模板

6. **API Keys:**
   - `GET /api-keys` - 密钥列表
   - `POST /api-keys` - 创建密钥
   - `DELETE /api-keys/:id` - 删除密钥
   - `GET /api-keys/:id/stats` - 密钥使用统计

7. **仪表板数据:**
   - `GET /dashboard/overview` - 概览数据
   - `GET /dashboard/device-usage` - 设备使用趋势
   - `GET /dashboard/app-usage` - 应用使用统计
   - `GET /dashboard/billing-stats` - 消费统计

8. **余额看板:**
   - `GET /billing/balance-trend` - 余额趋势
   - `GET /billing/consumption-forecast` - 消费预测
   - `GET /billing/alerts` - 账户预警
